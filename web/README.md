# Веб-интерфейс Pet SSL - Сокращение ссылок

Современный веб-интерфейс для сервиса сокращения ссылок, доступный по адресу `http://localhost:8080`.

## Функциональность

### Основные возможности:
1. **Сокращение ссылок** - введите любой URL, укажите название и пользовательский код (опционально)
2. **Просмотр созданных ссылок** - список последних сокращённых ссылок с статистикой кликов
3. **Статистика** - общее количество ссылок, кликов и кликов за сегодня
4. **Копирование ссылок** - одним кликом скопировать сокращённую ссылку в буфер обмена

### Технические особенности:
- Адаптивный дизайн (работает на мобильных устройствах и десктопах)
- Интуитивно понятный пользовательский интерфейс
- Интеграция с REST API бэкенда
- Уведомления о действиях пользователя
- Демо-режим для тестирования без аутентификации

## Использование

### 1. Запуск сервиса
```bash
docker-compose up -d
```

### 2. Откройте в браузере:
```
http://localhost:8080
```

### 3. Создание короткой ссылки:
1. Введите URL в поле "Введите URL"
2. (Опционально) Укажите название и пользовательский код
3. Нажмите кнопку "Сократить"
4. Скопируйте полученную короткую ссылку

### 4. Использование короткой ссылки:
- Откройте короткую ссылку в браузере для редиректа
- Просматривайте статистику кликов в разделе "Последние ссылки"

## Интеграция с API

Веб-интерфейс использует следующие API endpoints:

- `POST /api/v1/auth/register` - регистрация пользователя
- `POST /api/v1/auth/login` - аутентификация
- `POST /api/v1/links` - создание короткой ссылки
- `GET /{code}` - редирект по короткому коду
- `GET /api/v1/links/{code}/stats` - статистика ссылки

## Настройка

По умолчанию используется демо-токен. Для реального использования:
1. Реализуйте форму входа/регистрации
2. Сохраняйте JWT токен в localStorage
3. Передавайте токен в заголовках запросов

### Кастомизация
- **Стили**: редактируйте `style.css`
- **Логика**: редактируйте `app.js`
- **Верстка**: редактируйте `index.html`

## Устранение неполадок

### Проблема: Веб-интерфейс не загружается
1. Проверьте, запущены ли контейнеры: `docker-compose ps`
2. Проверьте логи приложения: `docker-compose logs app`
3. Убедитесь, что порт 8080 не занят

### Проблема: API запросы не работают
1. Проверьте, что бэкенд запущен и здоров
2. Проверьте CORS настройки (если фронтенд на другом домене)
3. Проверьте токен аутентификации

### Проблема: Статические файлы не загружаются
1. Убедитесь, что папка `web` существует в корне проекта
2. Проверьте права доступа к файлам
3. Пересоберите контейнер: `docker-compose build app`
