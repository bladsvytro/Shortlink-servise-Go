# Pet SSL — сервис сокращения ссылок

Высокопроизводительный, масштабируемый сервис сокращения ссылок с веб-интерфейсом, построенный на Go и PostgreSQL.

## 🚀 Быстрый старт

### Локальный запуск (Docker Compose)

Самый простой способ запустить весь стек (бэкенд, базу данных, Redis) с веб-интерфейсом:

1. **Клонируйте репозиторий**
   ```bash
   git clone <repository-url>
   cd Pet_SSL
   ```

2. **Настройте переменные окружения**
   ```bash
   cp .env.example .env.local
   # При необходимости отредактируйте .env.local
   ```

3. **Запустите все сервисы**
   ```bash
   docker-compose up -d
   ```

4. **Откройте веб-интерфейс**
   - Веб-интерфейс: http://localhost:8080
   - API: http://localhost:8080/api/v1
   - Health check: http://localhost:8080/health

5. **Остановка сервисов**
   ```bash
   docker-compose down
   ```

### Локальный запуск (без Docker)

Если вы предпочитаете запускать сервисы вручную:

1. **Установите зависимости**
   - Go 1.25+
   - PostgreSQL 14+
   - Redis (опционально)

2. **Настройте базу данных**
   ```bash
   createdb url_shortener
   # или используйте Docker для PostgreSQL:
   docker run --name postgres -e POSTGRES_PASSWORD=postgres -e POSTGRES_DB=url_shortener -p 5432:5432 -d postgres:15
   ```

3. **Запустите миграции**
   ```bash
   make migrate
   ```

4. **Запустите сервер**
   ```bash
   make run
   # или с hot reload для разработки:
   make dev
   ```

## ✨ Возможности

- **Сокращение ссылок** – создание коротких алиасов для длинных URL
- **Перенаправление** – быстрое перенаправление с отслеживанием кликов
- **Веб-интерфейс** – удобный web‑интерфейс для создания ссылок и просмотра статистики
- **Аутентификация** – регистрация и вход по JWT токенам
- **Статистика** – агрегированная статистика по всем ссылкам пользователя
- **Кастомные коды** – возможность задавать свой короткий код
- **Истечение срока** – установка времени жизни ссылки
- **Мониторинг здоровья** – встроенный эндпоинт `/health`
- **Структурированное логирование** – логирование через Zap

## 🛠 Технологический стек

- **Язык**: Go 1.25+
- **HTTP фреймворк**: Стандартная библиотека `net/http`
- **База данных**: PostgreSQL с ORM GORM
- **Кэширование**: Redis (опционально)
- **Аутентификация**: JWT с bcrypt хешированием паролей
- **Логирование**: Zap для структурированного логирования
- **Конфигурация**: Viper + godotenv
- **Контейнеризация**: Docker + Docker Compose
- **Фронтенд**: HTML/CSS/JavaScript (статический)
- **Тестирование**: Go testing package

## 🔌 API документация

### Health Check
```
GET /health
```
Возвращает `OK`, если сервис работает.

### Аутентификация

#### Регистрация
```
POST /api/v1/auth/register
```
**Запрос:**
```json
{
  "email": "user@example.com",
  "password": "securepassword",
  "name": "Имя пользователя"
}
```

#### Вход
```
POST /api/v1/auth/login
```
**Запрос:**
```json
{
  "email": "user@example.com",
  "password": "securepassword"
}
```
**Ответ:** содержит JWT токен и данные пользователя.

#### Получение информации о текущем пользователе
```
GET /api/v1/auth/me
```
Требует заголовок `Authorization: Bearer <token>`.

### Создание короткой ссылки
```
POST /api/v1/links
```
Требует аутентификацию.

**Запрос:**
```json
{
  "url": "https://example.com/very/long/url/path",
  "custom_code": "mycode",
  "title": "Пример страницы",
  "description": "Описание ссылки",
  "expires_at": "2026-12-31T23:59:59Z"
}
```

**Ответ:**
```json
{
  "short_code": "mycode",
  "short_url": "http://localhost:8080/mycode",
  "original_url": "https://example.com/very/long/url/path",
  "created_at": "2026-04-19T15:50:37.378421Z"
}
```

### Перенаправление на оригинальный URL
```
GET /{code}
```
Перенаправляет на оригинальный URL (HTTP 302). Увеличивает счётчик кликов.

### Статистика по конкретной ссылке
```
GET /api/v1/links/{code}/stats
```
**Ответ:**
```json
{
  "short_code": "mycode",
  "original_url": "https://example.com/very/long/url/path",
  "click_count": 42,
  "last_clicked_at": "2026-04-19T15:51:03.570469Z",
  "created_at": "2026-04-19T15:50:37.378421Z",
  "expires_at": null,
  "is_active": true
}
```

### Статистика пользователя
```
GET /api/v1/stats
```
Требует аутентификацию. Возвращает агрегированную статистику по всем ссылкам пользователя.

**Ответ:**
```json
{
  "total_links": 5,
  "total_clicks": 120,
  "today_clicks": 3
}
```

### Список ссылок пользователя
```
GET /api/v1/links
```
Требует аутентификацию. Возвращает список всех ссылок, созданных текущим пользователем, с пагинацией.

**Параметры запроса:**
- `page` (опционально, по умолчанию 1) – номер страницы
- `per_page` (опционально, по умолчанию 20) – количество ссылок на странице

**Ответ:**
```json
{
  "links": [
    {
      "short_code": "abc123",
      "original_url": "https://example.com",
      "title": "Пример",
      "click_count": 42,
      "created_at": "2026-04-19T15:50:37.378421Z",
      "expires_at": null
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 5,
    "total_pages": 1
  }
}
```

### Управление API ключами

API ключи позволяют программатически взаимодействовать с API без использования JWT токена.

#### Создание API ключа
```
POST /api/v1/apikeys
```
Требует аутентификацию.

**Запрос:**
```json
{
  "name": "Мой ключ",
  "expires_at": "2026-12-31T23:59:59Z"
}
```
**Ответ:** содержит сгенерированный ключ (отображается только один раз) и его метаданные.

#### Список API ключей
```
GET /api/v1/apikeys
```
Требует аутентификацию. Возвращает список API ключей пользователя (без самих ключей).

#### Удаление API ключа
```
DELETE /api/v1/apikeys/{id}
```
Требует аутентификацию. Удаляет API ключ по его ID.

### Управление доменами

Пользователи могут добавлять собственные домены для сокращения ссылок.

#### Создание домена
```
POST /api/v1/domains
```
Требует аутентификацию.

**Запрос:**
```json
{
  "domain_name": "my.domain.com"
}
```
**Ответ:** созданный домен со статусом `pending_verification`.

#### Верификация домена
```
POST /api/v1/domains/{id}/verify
```
Требует аутентификацию. Запускает проверку DNS записи (TXT) для подтверждения владения доменом.

#### Список доменов
```
GET /api/v1/domains
```
Требует аутентификацию. Возвращает список доменов пользователя.

#### Удаление домена
```
DELETE /api/v1/domains/{id}
```
Требует аутентификацию. Удаляет домен.

### Админ-эндпоинты

Доступны только пользователям с флагом `is_admin = true`.

#### Список пользователей
```
GET /api/v1/admin/users
```
Возвращает список всех пользователей с пагинацией.

#### Список всех ссылок
```
GET /api/v1/admin/links
```
Возвращает все ссылки в системе с пагинацией.

#### Статистика системы
```
GET /api/v1/admin/stats
```
Возвращает агрегированную статистику по всей системе: общее количество пользователей, ссылок, кликов и т.д.

## 🌐 Веб-интерфейс

Веб-интерфейс доступен по адресу http://localhost:8080 и предоставляет:

- **Форму создания коротких ссылок** – введите URL, заголовок и опциональный кастомный код
- **Отображение созданной ссылки** – с кнопкой копирования в буфер обмена
- **Карточку статистики** – общее количество ссылок, кликов и кликов за сегодня
- **Автоматическую аутентификацию** – демо-пользователь создаётся автоматически при первом посещении
- **Адаптивный дизайн** – корректно отображается на мобильных устройствах
- **Регистрация и вход** – модальные окна для регистрации и входа в систему
- **Админ-панель** – для администраторов доступна панель управления пользователями, ссылками и статистикой

### Полный гайд запуска

#### 1. Запуск с Docker Compose (рекомендуется)

```bash
# Клонируйте репозиторий
git clone <repository-url>
cd Pet_SSL

# Настройте переменные окружения
cp .env.example .env.local
# При необходимости отредактируйте .env.local

# Запустите все сервисы
docker-compose up -d

# Откройте веб-интерфейс
# Веб-интерфейс: http://localhost:8080
# API: http://localhost:8080/api/v1
# Health check: http://localhost:8080/health
```

#### 2. Запуск без Docker (для разработки)

```bash
# Установите зависимости
# - Go 1.25+
# - PostgreSQL 14+
# - Redis (опционально)

# Настройте базу данных
createdb url_shortener
# или используйте Docker для PostgreSQL:
docker run --name postgres -e POSTGRES_PASSWORD=postgres -e POSTGRES_DB=url_shortener -p 5432:5432 -d postgres:15

# Настройте переменные окружения
cp .env.example .env.local
# Отредактируйте .env.local, указав правильные параметры подключения к БД

# Запустите миграции
make migrate

# Запустите сервер
make run
# или с hot reload для разработки:
make dev
```

#### 3. Создание первого администратора

По умолчанию администраторы создаются вручную через базу данных. Выполните SQL-запрос:

```sql
INSERT INTO users (email, password_hash, name, is_admin)
VALUES ('admin@example.com', '$2a$10$...', 'Admin', true);
```

Пароль должен быть хеширован с помощью bcrypt. Вы можете использовать утилиту `bcrypt` или создать пользователя через API:

```bash
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com","password":"securepassword","name":"Admin"}'
```

Затем обновите запись в БД, установив `is_admin = true`:

```sql
UPDATE users SET is_admin = true WHERE email = 'admin@example.com';
```

### Доступ к админ-панели

1. **Войдите в систему** под учётной записью с флагом `is_admin = true`.
2. **Нажмите кнопку "Админка"** в правом верхнем углу веб-интерфейса.
3. **Используйте вкладки** для управления:
   - **Пользователи** – просмотр, удаление пользователей.
   - **Ссылки** – просмотр всех ссылок в системе.
   - **Статистика** – общая статистика системы (количество пользователей, ссылок, кликов).

Админ-панель предоставляет интерфейс для выполнения основных административных задач без необходимости использования API напрямую.


### Kubernetes

Манифесты и Helm чарты находятся в `deployments/kubernetes/` (планируется к реализации).

### Production рекомендации

1. **Переменные окружения** – все чувствительные значения (JWT secret, пароли БД) должны быть установлены безопасно, не коммититься в репозиторий.
2. **База данных** – используйте managed PostgreSQL сервис или запускайте PostgreSQL с репликацией и бэкапами.
3. **Redis** – включите Redis для кэширования и ограничения запросов (установите `REDIS_ENABLED=true`).
4. **SSL/TLS** – разместите сервис за reverse proxy (nginx, Traefik) с HTTPS.
5. **Мониторинг** – используйте эндпоинт `/health` для health checks, рассмотрите добавление метрик Prometheus.
6. **Масштабирование** – сервис stateless; масштабируйте горизонтально, запуская несколько инстансов за балансировщиком нагрузки.
